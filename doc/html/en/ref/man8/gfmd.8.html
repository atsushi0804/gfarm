gfmd

Name

gfmd(8) -- Gfarm metadata server

Synopsis

gfmd [options]

DESCRIPTION

gfmd is a Gfarm filesystem metadata server for accessing filesystem metadata of the Gfarm filesystem. gfmd is needed to be running on a metadata server node in advance.

It is possible for gfmd to be executed as a non-privileged user process although only the user that executes the gfmd can be authenticated. Other users cannot use it.

In order to execute gfmd as a non-privileged user process, a configuration file for a user needs to be specified by the -f option. At least, the port number used by gfmd should be altered to a non-privileged port number by metadb_serverport statement in the configuration file or the -p option.

In this case, the altered port number of gfmd should be specified by the metadb_serverport statement in ~/.gfarmrc or %%SYSCONFDIR%%/gfarm.conf on a client node.

OPTIONS

-P pid-file

specifies a file name which records the process ID of gfsd.

-d

specifies the debug mode. With the -d option, gfmd is running as a foreground process not a daemon.

-f config-file

specifies a configuration file that is read instead of the default configuration file.

-p port

specifies a port number used by gfmd. By default, 601 is used.

Default port requires the root privilege, although privileged ports are not necessary to execute gfmd.

-s syslog-facility

specifies a syslog facility to report errors by gfmd. By default, local0 is used.

-u

specifies a syslog facility to report errors by gfmd. By default, local0 is used.

-?

displays a list of command options.

FILES

%%SYSCONFDIR%%/gfarm.conf

configuration file
