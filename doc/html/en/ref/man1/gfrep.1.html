gfrep

Name

gfrep -- replicate a Gfarm file

Synopsis

gfrep -H hostfile gfarm-URL

gfrep -D domainname gfarm-URL

gfrep -I fragment-index [-s src-node] -d dest-node gfarm-URL

DESCRIPTION

gfrep creates a replica of a Gfarm file gfarm-URL. Destination filesystem nodes can be specified by the hostfile or the domainname.

With the -I option, gfrep creates a replica of a specified file fragment fragment-index of a Gfarm file gfarm-URL on the filesystem node dest-node specified by the -d option. With the -s option, it is possible to explicitly specify the source node src-node where the file fragment is copied from. When the specified Gfarm file has only one fragment, it is not necessary to specify the -I option.

OPTIONS

-H hostfile

specifies a list of filesystem nodes that will be replicated to. hostfile consists of a filesystem node in each line. The total number of file fragments should be equal to the number of nodes listed in the hostfile.

if ``-'' is specified, standard input is used to read the host list.

-D domainname

specifies a domainname of filesystem nodes that will be replicated to.

-I fragment-index

specifies a file fragment. It is a fragment index or a section name describing its architecture.

-s src-node

specifies a source filesystem node from which the file fragment will be copied. This option needs to be specified with the -I option.

-d dest-node

specifies a destination filesystem node to which the file fragment will be copied. This option needs to be specified with the -I option.

-?

displays a list of command options.
