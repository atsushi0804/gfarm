gfarm_agent

Name

gfarm_agent -- Gfarm Filesystem Metadata Cache Server

Synopsis

gfarm_agent [options]

DESCRIPTION

gfarm_agent is a cache server for a Gfarm metadata server that can be shared by multiple applications on the same machine. The cache server greatly improves performance and response time for a Gfarm file system by hiding network latency to a distant metadata server, and by preventing performance degradation when thousands of files are stored in the Gfarm file system.

To use the cache server, execute gfarm_agent on a client node or a compute node, and define environment variables output by gfarm_agent.

% eval `gfarm_agent`

Output environment variables are GFARM_AGENT_SOCK and GFARM_AGENT_PID. GFARM_AGENT_SOCK specifies a unix-domain socket that gfarm_agent is listening to. GFARM_AGENT_PID specifies a process ID of gfarm_agent.

gfarm_agent is running in the background. When defining the above environment variables, running gfarm_agent can be shared.

Any number of gfarm_agents can be running on different nodes (or even on the same node) at the same time. Consistency among multiple cache servers and a metadata server is maintained.

To terminate the cache server, send a TERM signal.

% kill $GFARM_AGENT_PID

OPTIONS

-s

generates Bourne shell commands.

-c

generates C-shell commands.

-d

specifies the debug mode. gfarm_agent is running in the foreground not in the background. Note that 'eval `gfarm_agent`' cannot work since it runs in the foreground.

-f config-file

specifies a configuration file that is read instead of the default configuration file.

-P pid-file

specifies a file name which records the process ID of gfarm_agent.

-S syslog-facility

specifies a syslog facility to report errors by gfarm_agent. By default, local0 is used.

-v

displays verbose error messages in authentication.

-?

displays a list of command options.

FILES

/tmp/.gfarm-XXXXXX/agent.<ppid>

a unix-domain socket that can be connected by the user that executes gfarm_agent only. It is automatically removed when gfarm_agent exits by receiving a TERM signal.
